ProGet supports Amazon S3 and Microsoft Azure Blob storage, giving you the ability to store all your packages in the cloud. ProGet's cloud package stores provides organizations with major benefits, including endless scalability and easy disaster recovery.

This feature is available in paid and trial ProGet editions.

Scale when you need with cloud package stores; this takes the guess work out of your future storage needs. By storing packages on the cloud, you no longer have to worry about maintaining, installing, or provisioning additional storage capacity.

Disaster recovery is made simple; both Amazon S3 and Azure Blob storage offer redudancy simplicity, meaning that you don't have to worry about complex, disaster recovery plans for large amounts of package files.

Configure Feed Package Store

Cloud storage may be configured on a feed-by-feed basis, giving you the flexibility to choose between different option for your packages.

Once a storage option is selected, you need to specify and configure the change. Note that the default is a file system directory.
