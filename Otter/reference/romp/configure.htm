The configure command runs a configuration plan first in collection mode, and again in execution mode if there was any drift detected.

Usage

romp collect <plan.otter> [--simulate]

The plan.otter argument specifies the file name of an OtterScript configuration plan.

The --simulate flag instructs Romp to perform the execution in simulation mode.

Behavior

Romp will perform the collection against the local machine and display the results on the console. If there was any drift detected, Romp will run the plan again in execution mode and attempt to remediate the drift. If the --simulate option is specified, no actual changes will be made to the machine during execution mode.

Examples

Execute the configuration plan contained in c:\plans\ensureWebsites.otter:

romp configure c:\plans\ensureWebsites.otter

Execute the configuration plan contained in c:\plans\ensureWebsites.otter, and load templates from c:\templates:

romp --lib=c:\templates configure c:\plans\ensureWebsites.otter

Execute the configuration plan contained in c:\plans\ensureWebsites.otter in simulation mode:

romp configure c:\plans\ensureWebsites.otter --simulate
