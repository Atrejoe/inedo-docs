The run command runs an orchestration plan.

Usage

romp run <plan.otter> [--simulate]

The plan.otter argument specifies the file name of an OtterScript orchestration plan.

The --simulate flag instructs Romp to perform the execution in simulation mode.

Behavior

Romp will perform the execution against the local machine and log to the console. If the --simulate option is specified, no actual changes will be made to the machine.

Examples

Execute the orchestration plan contained in c:\plans\createDevEnvironment.otter:

romp run c:\plans\createDevEnvironment.otter

Execute the orchestration plan contained in c:\plans\createDevEnvironment.otter, and load templates from c:\templates:

romp --lib=c:\templates run c:\plans\createDevEnvironment.otter

Execute the orchestration plan contained in c:\plans\createDevEnvironment.otter in simulation mode:

romp run c:\plans\ensureWebsites.otter --simulate
